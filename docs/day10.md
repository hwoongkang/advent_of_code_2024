# Day 10

https://adventofcode.com/2024/day/10

비교적 간단한 DFS/BFS 문제입니다.

## Part 1

0에서 시작하면 되고, +1씩 증가해서 9에서 끝나면 됩니다.

이떄 주의할 것이 9에 도달하면 무조건 1번만 셉니다.

즉 다른 방법으로 돌아서 같은 9에 도달했다 해도, 하나의 9에 도달한 것이기 때문에 1만큼만 정답에 관여해야 합니다.

=> 따라서 DFS로 풀면 됩니다.

지도의 크기를 n이라 하면 V~=O(n^2), E~=O(n^2)이므로 한 타일에 대해 DFS 할 때 복잡도는 O(V+E) = O(n^2)입니다.

모든 타일에 대해 DFS를 수행해주어야 하므로 (0에서 시작할 때만 하긴 하겠지만 일단 worst case로 추정하면) O(n^4)입니다.

입력의 n은 n~=50으로 충분합니다.

## Part 2

이번에는 파트 1의 제약조건이 조금 바뀌어서, 서로 다른 모든 경로를 다 세어야 합니다.

약간의 dp 개념을 도입해서, (r, c)위치에 있는 높이 h의 타일까지 도달하는 경로의 개수를 계속 저장하면 됩니다.

BFS로 구현 가능합니다.

BFS도 O(V+E)는 변함없으므로 O(n^4)에 가능합니다.
